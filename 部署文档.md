# ai-chart 部署文档

## Q：我到底是只把打包后的 `dist` 放服务器，还是把整个项目放服务器？

A：这次建议把**整个项目**放到服务器。

因为现在生产环境需要两部分：

1. 前端静态文件：`dist`
2. Node API 服务：`service/ai-api-server.mjs`

nginx 负责访问 `dist`，Node 服务负责读取服务器上的 API Key 并请求 DeepSeek / SiliconFlow。

如果只上传 `dist`，页面可以打开，但 `/api/chat-process`、`/api/session` 这些接口没有 Node 服务处理，就会无法正常聊天。

推荐服务器目录：

```bash
/home/web/ai-chart
```

最终结构大概是：

```txt
/home/web/ai-chart/package.json
/home/web/ai-chart/.env
/home/web/ai-chart/service/ai-api-server.mjs
/home/web/ai-chart/src
/home/web/ai-chart/vite.config.ts
/home/web/ai-chart/dist
```

## Q：完整部署步骤是什么？

### 第一步：本地确认要上传的内容

本地项目目录：

```txt
d:\project\zt\ai\ai-chart
```

上传到服务器：

```bash
/home/web/ai-chart
```

建议上传整个项目，但不要上传：

```txt
node_modules/
```

`dist/` 可传可不传，因为服务器上会重新打包生成。

需要保证服务器上有这些文件：

```txt
package.json
src/
public/
service/
vite.config.ts
index.html
.env
```

---

### 第二步：服务器安装 Node.js

登录服务器后检查 Node 版本：

```bash
node -v
```

建议 Node 版本为 18 或 20。

因为 `service/ai-api-server.mjs` 用到了 Node 内置的 `fetch`，Node 版本太低会启动失败。

---

### 第三步：上传项目到服务器

服务器创建目录：

```bash
mkdir -p /home/web/ai-chart
```

把本地整个项目上传到：

```bash
/home/web/ai-chart
```

注意不要套两层目录。

错误示例：

```txt
/home/web/ai-chart/ai-chart/package.json
```

正确示例：

```txt
/home/web/ai-chart/package.json
/home/web/ai-chart/service/ai-api-server.mjs
/home/web/ai-chart/src
/home/web/ai-chart/vite.config.ts
```

上传后检查：

```bash
cd /home/web/ai-chart
ls
```

应该能看到：

```txt
package.json
service
src
vite.config.ts
.env
```

---

### 第四步：确认服务器上的 `.env`

进入项目目录：

```bash
cd /home/web/ai-chart
```

编辑 `.env`：

```bash
nano .env
```

确认至少有这些配置：

```env
VITE_GLOB_API_URL=/api

VITE_GLOB_OPEN_LONG_REPLY=false
VITE_GLOB_APP_PWA=false

AI_PROVIDER=deepseek

DEEPSEEK_API_BASE_URL=https://api.deepseek.com
DEEPSEEK_API_KEY=你的key
DEEPSEEK_MODEL=deepseek-v4-flash
DEEPSEEK_CUSTOM_MODELS=deepseek-v4-flash,deepseek-v4-pro
```

如果使用 SiliconFlow，则改成：

```env
AI_PROVIDER=siliconflow

SILICONFLOW_API_BASE_URL=https://api.siliconflow.cn
SILICONFLOW_API_KEY=你的key
SILICONFLOW_MODEL=deepseek-ai/DeepSeek-V3
SILICONFLOW_CUSTOM_MODELS=deepseek-ai/DeepSeek-V3,deepseek-ai/DeepSeek-R1
```

保存后退出。

---

### 第五步：安装依赖

```bash
cd /home/web/ai-chart
npm install
```

如果服务器访问 npm 慢，可以使用国内镜像：

```bash
npm install --registry=https://registry.npmmirror.com
```

---

### 第六步：打包前端

```bash
npm run build
```

成功后会生成：

```txt
/home/web/ai-chart/dist
```

这个目录就是 nginx 要访问的前端静态目录。

---

### 第七步：临时启动 Node 服务测试

```bash
npm run api:start
```

如果正常，会看到类似输出：

```txt
[ai-api] listening on http://0.0.0.0:5001
[ai-api] provider=deepseek model=deepseek-v4-flash
```

这个方式只是测试用。关闭终端后服务会停止。

另开一个服务器终端测试：

```bash
curl http://127.0.0.1:5001/healthz
```

正常返回类似：

```json
{"code":200,"msg":"success","data":{"status":"ok","provider":"deepseek"},"rows":[]}
```

再测试：

```bash
curl -X POST http://127.0.0.1:5001/api/session
```

正常会返回 `auth`、`model`、`amodel`、`cmodels` 等信息。

测试没问题后，回到运行服务的终端，按：

```bash
Ctrl + C
```

先停掉临时服务。

---

### 第八步：用 pm2 后台启动 Node 服务

安装 pm2：

```bash
npm install -g pm2
```

进入项目目录：

```bash
cd /home/web/ai-chart
```

启动服务：

```bash
pm2 start service/ai-api-server.mjs --name ai-chart-api
```

查看状态：

```bash
pm2 list
```

看到 `ai-chart-api` 是 `online` 就说明启动成功。

查看日志：

```bash
pm2 logs ai-chart-api
```

正常会看到：

```txt
[ai-api] listening on http://0.0.0.0:5001
[ai-api] provider=deepseek model=deepseek-v4-flash
```

---

### 第九步：设置开机自启

先保存当前 pm2 进程列表：

```bash
pm2 save
```

然后执行：

```bash
pm2 startup
```

它会输出一条命令，类似：

```bash
sudo env PATH=$PATH:/usr/bin pm2 startup systemd -u root --hp /root
```

把它输出的那条命令复制出来执行一遍。

最后再执行：

```bash
pm2 save
```

这样服务器重启后，Node 服务会自动恢复。

---

### 第十步：修改 nginx 配置

nginx 配置参考：

```nginx
server {
    listen 8001;
    server_name 117.72.58.153;

    keepalive_requests 120;
    charset utf-8;

    root /home/web/ai-chart/dist;

    location / {
        index index.html;
        try_files $uri $uri/ @router;
    }

    location /api/ {
        proxy_http_version 1.1;

        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        proxy_set_header X-Forwarded-Port $server_port;

        proxy_buffering off;
        proxy_cache off;
        proxy_set_header Connection "";
        add_header X-Accel-Buffering no;

        proxy_connect_timeout 60s;
        proxy_send_timeout 1200s;
        proxy_read_timeout 1200s;

        proxy_pass http://127.0.0.1:5001;
    }

    location @router {
        rewrite ^.*$ /index.html last;
    }

    location ~ .*\.(html)$ {
        add_header Cache-Control no-cache;
        add_header Pragma no-cache;
    }

    error_page 500 502 503 504 /50x.html;
}
```

重点是这两处：

```nginx
root /home/web/ai-chart/dist;
```

```nginx
proxy_pass http://127.0.0.1:5001;
```

不要再写：

```nginx
proxy_pass https://web.pandarobot.chat/;
```

也不要直接写：

```nginx
proxy_pass https://api.deepseek.com;
```

正确链路是：

```txt
浏览器 /api/chat-process
  -> nginx
  -> http://127.0.0.1:5001/api/chat-process
  -> Node 服务读取 DEEPSEEK_API_KEY
  -> https://api.deepseek.com/v1/chat/completions
```

---

### 第十一步：检查并重载 nginx

检查配置：

```bash
nginx -t
```

如果显示类似：

```txt
syntax is ok
test is successful
```

再重载 nginx：

```bash
nginx -s reload
```

如果服务器用 systemctl 管理 nginx，也可以：

```bash
systemctl reload nginx
```

---

### 第十二步：浏览器访问验证

打开：

```txt
http://117.72.58.153:8001
```

按 F12 打开浏览器开发者工具，看 Network。

应该看到请求：

```txt
/api/session
/api/system/model/modelList
/api/chat-process
```

它们请求的是你自己的服务器：

```txt
117.72.58.153:8001/api/...
```

不应该再出现：

```txt
web.pandarobot.chat
```

也不应该在浏览器请求里看到你的 DeepSeek API Key。

## Q：以后每次更新代码怎么做？

### 情况一：只改了前端代码

例如修改了：

```txt
src/
```

服务器执行：

```bash
cd /home/web/ai-chart
npm run build
nginx -s reload
```

一般只重新打包即可，nginx reload 不是必须，但执行一下更稳。

---

### 情况二：改了 Node 服务代码

例如修改了：

```txt
service/ai-api-server.mjs
```

服务器执行：

```bash
cd /home/web/ai-chart
pm2 restart ai-chart-api
```

---

### 情况三：改了 `.env` 里的 key 或模型配置

例如修改了：

```txt
DEEPSEEK_API_KEY
DEEPSEEK_MODEL
AI_PROVIDER
```

改完后需要重启 Node 服务：

```bash
cd /home/web/ai-chart
pm2 restart ai-chart-api
```

原因是 Node 服务启动时读取 `.env`。

---

### 情况四：依赖变了

例如 `package.json` 变了。

服务器执行：

```bash
cd /home/web/ai-chart
npm install --registry=https://registry.npmmirror.com
npm run build
pm2 restart ai-chart-api
```

最后浏览器访问：

## Q：一句话总结？

A：上传整个项目到服务器。nginx 只访问项目里的 `dist`，Node 服务运行项目里的 `service/ai-api-server.mjs`。所以不是只传 `dist`，否则 Node 服务没有文件可启动。
